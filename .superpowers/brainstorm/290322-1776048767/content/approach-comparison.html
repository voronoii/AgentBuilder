UI 리디자인 접근 방식

Clay 디자인 토큰을 유지하면서 상업용 품질로 올리는 3가지 방법

A
A Clay Elevation — 테두리 제거 + 그림자 기반 추천

노드 테두리를 완전히 제거하고 Clay 디자인 시스템의 Elevation Level 1 (Clay shadow)으로 깊이감을 표현합니다. 노드 타입별로 헤더 배경색만 달리하여 구분합니다. Dify 스타일에 가장 가까우면서 Clay 톤을 유지합니다.

Language Model
gpt-4o · temperature 0.7
🤖 Agent
max iterations: 5
📄 Knowledge Base
top_k: 3 · threshold: 0.7

장점

  • 가장 세련되고 모던한 느낌
  • Clay 엘리베이션 체계와 완벽 일치
  • Dify와 유사한 프로페셔널 톤

단점

  • 노드 타입 구분이 미묘할 수 있음
  • 캔버스에서 노드가 묻힐 가능성
B
B Thin Border + Color Dot — 얇은 테두리 + 컬러 인디케이터

테두리를 1px oat 색상으로 통일하고, 노드 타입은 헤더 왼쪽의 작은 컬러 도트로 구분합니다. 깔끔하면서도 노드 경계가 명확합니다. 현재 스타일과 가장 부드럽게 전환 가능합니다.

Language Model
gpt-4o · temperature 0.7
Agent
max iterations: 5
Knowledge Base
top_k: 3 · threshold: 0.7

장점

  • 노드 경계가 확실히 보임
  • 기존 코드에서 변경량 적음
  • 심플하고 실용적

단점

  • 시각적 임팩트가 약함
  • A보다 덜 모던한 느낌
C
C Left Accent — 왼쪽 컬러 바 + 미니멀 바디

노드 왼쪽에 3px 컬러 악센트 바를 두고 나머지는 흰색 배경 + 그림자. n8n이나 일부 IDE 스타일과 유사합니다. 타입 구분이 직관적이면서 깔끔합니다.

Knowledge Base

장점

  • 타입 구분이 한눈에 보임
  • 깔끔하고 정돈된 느낌

단점

  • n8n과 너무 비슷해 보일 수 있음
  • 왼쪽 바가 다소 기계적인 느낌
